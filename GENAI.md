# GenAI Usage Policy — AgroGuardAI

This document discloses how Generative AI tools are used
in this project, in full compliance with the NLnet NGI Zero
Commons Fund GenAI policy (effective December 8, 2025).

---

## General Stance

AgroGuardAI uses Claude Sonnet (Anthropic) to assist with:
- Code scaffolding and boilerplate generation
- Documentation writing and structuring
- Prompt engineering for the agricultural AI system
- English language expression (founder is not a native speaker)

All AI-assisted outputs are reviewed, corrected, and validated
by the human team before being committed to the codebase.
The founder actively directed, corrected, and rejected outputs
throughout the development process.

---

## Commit Convention

When a commit contains AI-assisted code, the commit message
includes the model and prompt:

```
Author: Muhammad Zakari with Claude Sonnet (Anthropic)
Date: [date]

feat: add multi-LLM provider router
Prompt: Build a unified API router that supports Gemini,
Claude, OpenAI and DeepSeek with the same interface.
Output: this commit
```

---

## What AI Did NOT Provide

The following originated entirely from the founder:

- The founding mission and agricultural problem framing
- The 90% accuracy validation result from KATDICT
- The institutional relationship with KATDICT
- The AIC 2026 acceptance
- All budget decisions and figures
- The Kano field work location
- The decision to include Starlink for connectivity
- The multi-LLM architecture decision
- The correction of all overstated competitive claims
- The Agrobotics hardware research-phase status
